114. 二叉树展开为链表

空白页

算法思路：

  1. 根据树的最终形态可以知道，树中任意一颗节点都没有左孩子。最终所成的链表状态与树的先序遍历相同，先序遍历的顺序为根左右
  2. 因此对于根节点而言，左子树任意节点在先序遍历中的顺序都是在根节点右子树的任意结点之前的。
  3. 所以可以考虑每次将根节点的左子树接成根节点的右子树，使用原左子树中的最右节点连接根节点的原右子树。
  4. 然后让root前进，即root = root->right
  5. 重复34两步，直至遍历完成整棵树。

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

剑指 Offer 28. 对称的二叉树

空白页

算法思路：
判断一棵树是否为对称的，可以把这个问题拆解成两一棵树的两个子树是否是镜像，于是这个问题从判断一棵子树是否对称的问题转换成了两颗树是否是镜像的问题。
判断不同条件，两树节点数不同和对应节点值不同.

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(n)O(n)O(n)

错误原因：当前节点的左右孩子不一定符合上一层的判断，
例如：

二叉查找树的定义

  1. 根结点只包含一个关键字
  2. 非空左子树的关键字值一定小于其子树根节点的关键字值
  3. 非空右子树的关键字值一定大于其子树根节点的关键字值
  4. 左、右子树仍然是二叉查找树

正确思路：记录下当前节点的上下界，如果在上下界内，说明此节点符合二叉搜索树的定义，初始时上下界分别为long型的最大值与最小值，之后对左子树更新最大值，值为root->val,对右子树更新最小值，值为root->val

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(n)O(n)O(n)

662. 二叉树最大宽度

空白页



算法思路：
本题，要求求二叉树的最大宽度，实际上是求在所有层中最左节点到最右节点的距离，从而联想到二叉树的顺序存储，找到最右节点的在顺序存储中的位置right，然后减去最左节点在顺序存储中的位置left，但是要注意是求宽度，因此实际上是right - left + 1。
采用层次遍历，按层次计算每层的宽度，为了存储节点在顺序存储中的位置，采用pair来记录节点和位置

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(n)O(n)O(n)

110. 平衡二叉树

空白页


算法思路：
使用一个bool变量is_balance来表示二叉树是否为平衡二叉树,为了减少对节点的遍历次数,在计算树的深度时,就对节点是否平衡做判断，步骤如下：

  1. 如果root == nullptr,|| is_balance == false 返回0,当树中有一个节点不平衡时，不用再计算树的深度
  2. 计算左右子树高度,如果左右子树高度之差大于1,is_balance = false;
  3. 返回当前层的深度 max(left_height, right_height + 1);

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(n)O(n)O(n)

226. 翻转二叉树

空白页

算法思路：
采用递归的思路，递归的遍历左右子树，然后交换当前节点的左右子树，然后返回当前节点。具体步骤如下：

  1. 如果当前节点为空，返回
  2. 递归左子树和右子树
  3. 交换当前节点的左右子树
  4. 返回当前节点

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(n)O(n)O(n)

572. 另一棵树的子树

空白页


算法思路：
定义一个辅助函数check来检测root和subRoot是否相同，在check中：

  1. 若root和subRoot都为空，则root和subRoot相同
  2. 若root和subRoot不同时为空，或者root和subRoot的值不相等，此时root和subroot不同
  3. 递归的检查root和subRoot的左子树和root和subRoot的右子树

在isSubtree中
先check(root, subRoot),如果为真，返回true；
否则递归的检查root->left，subRoot 或root->right, subRoot

复杂度：
若主树有n个节点，次树又m个节点
时间复杂度：O(nm)O(nm)O(nm)
空间复杂度：O(n)O(n)O(n)

112. 路径总和

空白页


算法思路：
增加一个辅助变量has_targetSum，初始为false，使用一个辅助函数dfs，接收root，sum，targetSum，其中sum是从根节点到当前节点的路径长度（不包括当前节点）
在dfs中：

  1. 如果root为空，直接返回
  2. 如果root为叶节点，且sum + root->val = targetSum,说明在树中存在这么一条路径，置has_targetSum为true
  3. 递归root的左右子树

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(n)O(n)O(n)

剑指 Offer 54. 二叉搜索树的第k大节点

空白页

算法思路：
我们知道，二叉搜索树的中序遍历序列是一个单调增的序列，如果寻找第k小的数，只需要在返回中序序列中的第k个值即可，但此题要求返回第k大的数字，我们可以从中序遍历出发，中序遍历的次序为左根右，这样得到的二叉搜索树的中序序列是一个单调增的序列，如果我们更改中序遍历的次序，变为右根左，那么此时所得到的二叉搜索树的中序序列是一个单调减的序列。
根据上述分析，直接返回右根左中序序列中的第k个值即可
这里采用非递归的中序遍历
复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(n)O(n)O(n)

  1. 如果被删除的节点是叶节点，直接将其删除
  2. 如果被删除的节点没有左孩子，更新当前节点为其左孩子
  3. 如果被删除的节点没有右孩子，更新当前节点为其右孩子
  4. 如果被删除的节点有左右孩子，那么寻找该节点的前驱或后继，将该节点的值变更为其前驱或后继的节点的值，然后删除其前驱或后继
