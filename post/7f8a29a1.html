1.3文件系统层次标准与文件操作

1.3.1文件系统层次标准

/home系统默认的用户家目录，~代表当前用户家目录
/etc放置系统主要配置文件，在1.1中更换源就是在/etc/apt/sources.list中替换的
/bin与/usr/bin内容一致,/bin是/usr/bin的符号链接
usr-Unix Software Resource
which查询可执行程序的位置,cd是shell内置命令,例如：

dev – 设备以文件的形式存在这个目录中,cat – concatenate（串联，连接）,可查看cpuinfo
root-root用户的根目录
media 放置可删除设备，包括光盘、U盘
mnt – 挂载额外的存储设备
opt – 手动指定第三方软件放置的目录
tmp – 暂时放置文件的地方
var – 与系统的运行有关
lost+found – 针对ext2、ext3、ext4,当文件系统发生错误时，将一些遗失的片段放在这个目录下

1.4.1 打包解压

scp 本地路径 用户名@地址:linux路径从Windows上传文件到linux

tar文件
  • *.tar tar程序打包的文件，没有压缩
  • *.tar.gz tar程序打包的文件，并且经过gzip的压缩
  • *.tar.bz2 tar程序打包的文件，并且经过bzip2的压缩
  • *.tar.xz tar程序打包的文件，并且经过xz的压缩
tar文件的解压与压缩
  • -f指定文件,或直接在最后加f,
tar.gz
  • tar –x（extract） 解包或解压缩
  • tar –c 创建打包文件
  • tar –v压缩或者解压过程中，将正在处理的文件名打印
  • tar –z 通过gzip进行压缩/解压缩文件最好为*.tar.gz
  • -C 指定目录,指定解压目录
tar.bz2
  • tar -j通过bzip2进行压缩/解压缩文件最好为*.tar.bz2
tar.xz
  • tar –J 通过xz进行压缩/解压缩文件最好为*.tar.xz
  • –p 保留备份数据的原本权限和属性

1.4.3 环境变量的设置

  • echo 打印 显示
  • $ - PATH前面加$表示后面接的是变量
  • PATH – 这个变量是由一堆目录组成目录之间用 ：来隔开

直接执行hello程序的方法

  • sudo cp ./hello /usr/local/bin,将hello程序复制到usr/local/bin下
  • 临时添加环境变量：export PATH=/home/sjy/test/hello:$PATH
  • 在当前目录中添加环境变量,修改.bashrc文件,添加PATH=$PATH:/home/sjy/test/hello,再执行source ./.bashrc
  • 永久修改环境变量：修改/etc/environment，在最后添加:/home/sjy/test/hello
