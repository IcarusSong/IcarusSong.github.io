线索二叉树

基本概念

在一个二叉树中，所有指针的数量为2n，其中空指针的数量为n+1,为了利用这些空指针，有如下操作:
如果一个节点左孩子指针为空，令其左孩子指针为该节点的前驱；
如果一个节点右孩子指针为空，令其右孩子指针为该节点的后继。

为了方便对线索二叉树的操作(遍历序列中的第一个节点得前驱和最后一个节点的后继也能线索化)，加入一个头节点

(1) 当p->right_thread == true 时，p->right即为其后继
(2) 当p->right_thread == false 时，p的后继为其右子树的最左节点

求中序线索二叉树中节点p的先序顺序的后继节点p*

  1. p的左子树不空时，p的左儿子p->left即为p*；
  2. p的左子树为空但右子树不空时，p的p->right为p*;
  3. p的左右子树均空时，右线索序列中第一个有右孩子结点的右儿子或头结点即为p*

二叉树的（中序）线索化-递归

算法思想：
二叉树线索化，只需要按照某种次序遍历二叉树，在遍历过程中用线索取代空指针即可。为此，附设一个指针pre，pre始终指向刚刚访问过的节点，而指针p指示当前正在访问的节点。显然，节点pre是节点p的前驱，节点p是节点pre的后继。
实现步骤：

  1. 如果二叉链表root为空，则返回；否则，
  2. 对root的左子树建立线索；
  3. 对根节点root建立线索
    1. 若root没有左孩子，则为root加上前驱线索；
    2. 若root没有右孩子，则将root右线索标志置为true;
    3. 若节点pre右线索标志位true，则为pre加上后继线索；
    4. 令pre指向刚刚访问的节点root
  4. 对root的右子树建立线索。

哈夫曼树与哈夫曼编码

哈夫曼编码

哈夫曼编码是一种可变长度编码方式，用于将字符等数据压缩存储或传输。它的基本思想是根据字符出现频率构建一棵哈夫曼树，然后给每个字符分配一个唯一的二进制编码，使得出现频率高的字符获得短的编码，出现频率低的字符获得长的编码。

哈夫曼编码的构造过程如下：

  1. 统计每个字符在文本中出现的次数，并按照出现次数从小到大排序。
  2. 将出现次数最少的两个字符作为叶子节点构建一棵二叉树（称为哈夫曼树），其中较小的字符为左子节点，较大的字符为右子节点，其父节点的权值为这两个字符的权值之和。
  3. 将构建好的哈夫曼树作为一个整体，将其权值作为新的节点权值，重新排序后插入到剩余的节点中，重复以上步骤，直到只剩下一个根节点。
  4. 从根节点开始遍历哈夫曼树，对左子树赋值“0”，对右子树赋值“1”，得到每个字符的编码。
    使用哈夫曼编码进行压缩时，只需要将文本中的每个字符替换为对应的编码即可。由于出现频率高的字符获得短的编码，因此压缩后的数据量会大幅减少。

需要注意的是，哈夫曼编码的解码过程需要先构建哈夫曼树，然后按照编码逐个遍历该树，找到对应的字符即可。

并查集

并查集（Disjoint Set）是一种常用的数据结构，主要用于处理一些不相交集合的合并及查询问题。它支持两个基本操作：

查找（Find）：查找元素所在的集合。
合并（Union）：将两个不相交的集合合并成一个集合。
并查集通常用于解决连通性问题，例如判断图中是否存在环，或者求图的连通分量等。它可以用来维护一组不相交的集合，每个集合内部的元素之间互相连通，而不同集合之间没有连接。实现时，通常使用一个数组来表示每个元素所属的集合，这个数组被称为“父节点数组”或“祖先数组”。
