19. 删除链表的倒数第 N 个结点(2009年真题)

空白页

算法思路：

  1. 采用快慢指针
  2. 快指针先走n步
  3. 快慢指针同时前进，当快指针走到空节点时，慢指针的next指向倒数第n个节点
  4. 删除节点
    复杂度：
    时间复杂度：O(n)O(n)O(n),空间复杂度O(1)O(1)O(1)

剑指 Offer 52. 两个链表的第一个公共节点(2012年真题)

空白页


算法思路：

  1. 计算出两条链表的长度和链表长度之差diff
  2. 让较长的链表先走diff步
  3. 两条链表同时前进，若节点相同则返回，否则继续前进
  4. 没有找到相同节点，返回NULL

筛选链表(2015年真题)

空白页

算法思路：
开辟一个长度为n的大数组h，每次遍历当前元素时，检查h[|cuur->val|]是否为0，如果为0说明该绝对值是第一次遍历，如果不为0，说明该绝对值不是第一次遍历到了，应该将该节点删除
复杂度：
时间复杂度O(m)O(m)O(m),空间复杂度O(n)O(n)O(n)

143. 重排链表(2019年真题)

空白页

算法思路:

  1. 利用快慢找到链表的中间节点
  2. 将链表的后半段逆置
  3. 将前半段链表与逆置后的后半段链表合并

复杂度
时间复杂度O(n)O(n)O(n)
空间复杂度O(1)O(1)O(1)

2. 两数相加

空白页

算法思路：

  1. 两条链表从头节点到尾节点是按照最低位到最高位的顺序
  2. 模拟纸笔加法，使用t来代表当前位的和，并且在计算下一位时，t变为进位
    复杂度：
    时间复杂度：O(n)O(n)O(n),空间复杂度：O(1)O(1)O(1)

21. 合并两个有序链表

空白页

算法思路：

  1. 类似与二路归并排序，但是不需要借助辅助空间，只需要使用一个prev节点
  2. 每次让prev->next指向两条链表中较小的节点，然后prev前进
  3. 等到遍历完至少一条链表之后，开始进行扫尾工作,将prev->next指向剩余的非空链表

复杂度：
时间复杂度：O(n+m)O(n+m)O(n+m)
空间复杂度：O(1)O(1)O(1)

83. 删除排序链表中的重复元素

空白页

算法思路：
判断当前节点的值是否和下一个节点的值相等，若相等，则让当前节点指向下一节点的下一个节点，否则使得当前指针前进

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

82. 删除排序链表中的重复元素 II

空白页

算法思路：
因为链表已经是有序状态，所以重复的元素一定是连续的，使用三个节点prev，curr，next

  1. 增加虚拟头结点dummy，prev初始指向dummy，curr指向head，而next为curr->next
  2. 每次比较curr和next的值，如果相等，next向后走，知道走到curr的值和next的值不相等的节点，此时，从curr到next之前的所有节点都是重复元素，都应该删除，让prev->next,curr = next
  3. 如果比较curr和next的值，二者不等，prev和curr前进

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

24. 两两交换链表中的节点

空白页

算法思路：
每次在两两交换节点first和second时，保存其前节点prev，保证链表不断链，随后交换first和second的指向，并将prev的指向更新为second，随后用first更新prev

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

876. 链表的中间结点

空白页

算法思路：
使用快慢指针，快指针走两步，慢指针走一步，当快指针指向空，或者快指针的下一节点指向空时，慢指针恰指向中间节点

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

2095. 删除链表的中间节点

空白页

算法思路：

  1. 增加一个虚拟头结点
  2. 采用快慢指针fast/slow,初始时slow = dummy,fast = head,这样可以保证找到的是中间节点的前驱节点
  3. fast每次走两步,slow每次走一步
  4. 删除中间节点

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

206. 反转链表

空白页

算法思路：
迭代法:

  1. 每次让curr指向prev,curr = curr->next,prev = curr,如此整条链表被反转

头插法：
每次将遍历的节点采用头插法插入到链表中,最后的链表为反转过的链表
复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

141. 环形链表

空白页


算法思路：
采用快慢指针，快指针走两步，慢指针走一步，如果链表中有环，则快慢指针一定能够相遇，否则不能相遇

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

142. 环形链表 II

空白页


算法思路：

  1. 先根据上一题的思路，判断链表是否有环路，如果有环路，将hasCycle置为true
  2. 在有环的情况下，计算环的长度cycleLength
  3. 将一条有环的链表先视为一条无环的链表，在无环链表中，尾节点的下一节点应该是空节点，而在有环链表中，尾节点的下一节点是环的入口
  4. 此时，可以采用寻找倒数第n个节点的思路，采用快慢指针找到环的入口，即在无环中找到第n个节点，因为有环，所以退出循环的条件应该有fast != NULL变为 fast == slow

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

剑指 Offer II 029. 排序的循环链表

空白页


算法思路：
本题一共有四种情况:

  1. 链表为空,根据插入值新建一个节点返回即可
  2. 链表只有一个节点,新建节点插入在头结点之后,保证还是循环链表即可
  3. 链表中有两个以上的节点,链表中存在比插入值大的数,将插入值插入在合适位置即可
  4. 链表中有两个以上的节点,但是链表中不存在比插入值大的数,此时将插入值插入在链表最大值之后

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

234. 回文链表

空白页

算法思路：

  1. 使用快慢指针找到中间节点的前一个位置
  2. 从中间节点开始反转链表，形成两个链表，此时，第一个链表的长度一定是小于等于第二个链表的长度的
  3. 从两个链表的表头开始遍历(循环结束条件为第一个链表不为空，这样可以应对奇数长度的链表)，判断节点值是否相同，如果不同返回false,相同就继续前进
  4. 完整遍历第一段链表后说明是回文链表返回true

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

61. 旋转链表

空白页


算法思路：
旋转链表是将后k个元素旋转到前面的元素之前，可以转换成寻找第倒数第n个节点问题

  1. 计算出链表长度,将k的值对链表长度取余，保证k是小于链表长度的
  2. 使用快慢指针寻找到倒数第k个节点的前驱节点
  3. 随后更改多个节点的next即可

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

86. 分隔链表

空白页

算法思路：

  1. 题目要求小于x的节点都在链表前面，大于等于x的节点在链表后面，于是考虑将链表分成两条链表small和large，small保存小于x的节点，large保存大于等于x的节点
  2. 遍历整条链表，每次根据当前节点的值，将该节点分配给small或large
  3. 拼接链表

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

1290. 二进制链表转整数

空白页

算法思路：
根据二进制的特点,链表是从高位到低位表示的,每一位的实际值应是2len−k2^{len - k}2len−k,len为链表长度,k为当前节点在链表中的位置(从1开始计算),把所有值相加即可

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)

92. 反转链表 II

  3. 连接反转之后的链表

25. K 个一组翻转链表

空白页


算法思路：
按照k个一组反转链表，难点在于使得链表不断链，于是last_tail指向上一段反转过的链表的尾节点,last_head指向当前要反转链表的头结点，在反转之后，使last_tail->next指向反转后的链表的头结点,用last_head的值更新last_tail

复杂度：
时间复杂度：O(n)O(n)O(n)
空间复杂度：O(1)O(1)O(1)
